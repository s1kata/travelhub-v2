# Деплой сайта Travel Hub на хостинг

## Какую папку заливать

Заливай на хостинг **всю папку проекта** `--main` (корень репозитория):

```
D:\работа\--main\
```

Содержимое должно оказаться в **корне сайта** (document root). Структура после загрузки:

```
/                    → index.php, .htaccess
/frontend/           → страницы, CSS, JS
/backend/            → API, config, payment
/data/               → кэш, БД SQLite (создаётся автоматически)
/vendor/             → зависимости Composer (обязательно!)
/.env                → создай вручную на хостинге, не коммить!
```

## Важно

### 1. Запустить Composer

На хостинге (или локально перед загрузкой) в корне проекта:

```bash
composer install --no-dev
```

Папка `vendor/` должна быть на сервере.

### 2. Создать `.env` на хостинге

Скопируй `.env.example` в `.env` и заполни. **Никогда не коммить .env с паролями!**

Для оплаты через приложение (Т-Касса) нужны:

```env
TINKOFF_TERMINAL_KEY=твой_ключ_из_ЛК_Т_Банка
TINKOFF_PASSWORD=твой_пароль_терминала
APP_URL=https://твой-домен.ru
API_URL=https://твой-домен.ru
```

### 3. В личном кабинете Т-Банка

Укажи **NotificationURL** для вебхука:

```
https://твой-домен.ru/api/payment-webhook
```

### 4. Права на папки

Папка `data/` должна быть доступна для записи (775 или 755):

```bash
chmod -R 755 data/
```

## API для мобильного приложения

После деплоя приложение будет обращаться к:

- `POST https://твой-домен.ru/api/create-payment` — создание платежа
- `GET https://твой-домен.ru/api/payment-status/:transactionId` — статус

В приложении TravelHubNew в `.env` задай:

```env
PAYMENT_PAGE_URL=https://твой-домен.ru
```
