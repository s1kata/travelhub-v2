# ⚡ Быстрое решение проблем производительности

## 🚨 Проблема: TTFB 7 секунд

### Решение 1: Включить кэширование (САМОЕ ВАЖНОЕ!)

Кэширование уже добавлено на главную страницу. Для других страниц добавьте в начало файла:

```php
<?php
require_once __DIR__ . '/../backend/components/page_cache.php';

// Проверяем кэш
if (PageCache::get()) {
    exit;
}

// Начинаем буферизацию
PageCache::start();
```

И в конце файла перед `</body>`:

```php
<?php PageCache::end(); ?>
```

**Результат**: TTFB уменьшится с 7 секунд до < 200ms после первого запроса.

## 🖼️ Проблема: Большие изображения (fon2.webp 4.3 МБ)

### Решение 2: Оптимизировать изображения

1. **Запустите скрипт оптимизации:**
```bash
php scripts/optimize_images.php
```

2. **Или вручную сожмите изображения:**
   - Используйте [TinyPNG](https://tinypng.com/) или [Squoosh](https://squoosh.app/)
   - Для `fon2.webp`: качество 70-75%, размер должен быть ~200-300 КБ
   - Для логотипов: используйте SVG или сожмите до < 50 КБ

3. **Добавьте lazy loading:**
```html
<img src="image.jpg" alt="Описание" loading="lazy" width="800" height="600">
```

**Результат**: Размер страницы уменьшится с 5+ МБ до < 2 МБ.

## 📝 Проблема: Медленная загрузка CSS

### Решение 3: Критический CSS уже добавлен

Критический CSS автоматически встраивается inline. Дополнительно:

1. **Минифицируйте CSS файлы:**
   - Используйте [CSS Minifier](https://www.cssminifier.com/)
   - Или инструменты командной строки

2. **Разделите CSS на критический и некритический:**
   - Критический (первый экран) - inline
   - Остальной - загружать асинхронно

## 🔧 Настройка кэша

В начале PHP файла (перед `require_once`):

```php
// Время жизни кэша (в секундах)
define('PAGE_CACHE_TTL', 3600); // 1 час

// Отключить кэш (для отладки)
// define('PAGE_CACHE_DISABLED', true);
```

---

## 🚨 Важно!

- Кэш автоматически не используется для авторизованных пользователей
- Очищайте кэш после обновления контента: `PageCache::clear()`
- Первый запрос будет медленным (генерация кэша), последующие - быстрые

---

## 📞 Если проблемы остались

1. Проверьте права доступа к `data/page_cache/`
2. Убедитесь, что OPcache включен на сервере
3. Проверьте логи PHP
4. Обратитесь в поддержку хостинга с данными о TTFB
