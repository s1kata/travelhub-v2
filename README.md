# Travel Hub v2

Редизайн сайта travelhub63.ru: та же структура страниц и бизнес-логика, новый визуальный слой и сокращённая лид-воронка.

## Что это

Копия рабочего проекта v1 (PHP + MySQL + Tailwind CDN) с новой темой оформления поверх существующей дизайн-системы. Все API, поиск туров (Tourvisor), оплата (Т-Банк через мобильный API-слой), офисы и админка работают как в v1.

## Новое в v2 (кратко)

- **Дизайн-система v2** (сгенерирована через UI/UX Pro): палитра deep navy `#1A1A40` + teal `#5DA9A4` + коралловый CTA `#FF6B6B`, шрифты Outfit (заголовки) и Work Sans (текст).
- **Тема** — `frontend/css/v2-theme.css` + `frontend/js/v2-theme.js`: подключаются глобально через `backend/components/design_system_head.php` и `backend/components/header.php`, отдельно — на главной и `tour-detail.php`.
- **Лид-воронка 3–4 клика**: карточка тура → кнопка «Забронировать» → страница тура с автоматически открытой формой (`open_booking=1`) → отправка заявки / оплата в модалке. Отдельной страницы оплаты нет.
- **Микровзаимодействия**: hover-подсветка кнопок и карточек, scroll-reveal секций, анимация модалки, кнопка «Наверх».
- Блок **«Оставить заявку менеджеру» не изменён** (по просьбе Никиты).

Полный список — в [CHANGELOG.md](CHANGELOG.md).

## Стек

- PHP 7.4+ (рекомендуется 8.1+), MySQL 5.7/8.0
- Apache + `.htaccess` (роутинг, редиректы, кэш-заголовки)
- Tailwind CDN + собственные CSS (design-system, redesign, v2-theme)
- Vanilla JS (карточки туров, модалка бронирования, платежный API)

## Деплой на Spaceweb (sweb.ru)

Хостинг полностью совместим с проектом:

| Требование проекта | Spaceweb |
|---|---|
| PHP | Поддерживаются 5.3–8.5, версия меняется в панели «Сайты → Конфигурация» (по умолчанию 8.1) |
| MySQL | 5.7 и 8.0, phpMyAdmin в панели |
| Apache + `.htaccess` | Да, полная поддержка (редиректы, mod_rewrite) |
| Cron | Да («Задачи по расписанию» в панели) — нужен для прогрева кэша и фидов |
| SSL | Бесплатные сертификаты в панели |

### Шаги

1. Залить содержимое проекта в корень сайта (`public_html`) по FTP/SSH или через файловый менеджер панели.
2. В панели выбрать **PHP 8.1+** для сайта.
3. Создать базу MySQL, импортировать дамп v1, прописать доступы в `.env` (см. `.env.example`).
4. Проверить, что `.htaccess` подхватился (главная открывается, редиректы работают).
5. Настроить cron (как в v1):
   - `backend/scripts/promo_tours_refresh.php` — 2 раза в сутки;
   - `clear_cache.php 10` — очистка старого кэша;
   - при необходимости `backend/scripts/tourvisor_background_update.php`.
6. Каталоги `data/` и `img/` должны быть доступны на запись.

## Структура

```
frontend/        страницы, css (v2-theme.css), js (v2-theme.js)
backend/         компоненты, API-прокси Tourvisor, платежный слой, скрипты
api/             мобильный API (оплата, авторизация приложения)
img/, export/    статика и фиды
.htaccess        роутинг и редиректы
```
